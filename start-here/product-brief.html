Product Brief

What the Commercial Intelligence Lab is — in one read

Start Here › Product Brief

Audience: investors, operators, and anyone who wants the whole picture without the machinery.

v0.1 · Hypothesis stage See the Decisions

The one-paragraph version

We took the Sigmoid Analytica CRM proposal and turned it from a slide-deck argument into a living decision system — a site where commercial strategy is broken into concrete decisions, scored with transparent, adjustable frameworks, and held to an explicit standard for what counts as evidence. It runs on real code with an automated test suite, so every number can be re-run and audited. Crucially, it is honest about what it does not yet know: nothing is presented as a validated fact.

Why it is built this way

A proposal is an argument made before the evidence is in. If we treat it as truth, we bake in untested assumptions about market, positioning, and pricing. So the project is governed by one rule set — the Commercial Intelligence Specification (CIS) — that treats the proposal as Hypothesis v0.1 and forces every recommendation through a fixed lifecycle before it can claim authority. Every claim carries a confidence grade: D (asserted) → C (illustrative) → B (substantiated) → A (validated). Today everything is C or D, by design.

The three decisions the whole thing exists to make

Decision Question Status
Market Entry Which segment should we target first for the CRM? Hypothesis (C)
Positioning CRM, Revenue Operations, or AI Readiness Platform? Hypothesis (C)
GTM Evidence Gate What evidence is required before we commit to GTM? Gate defined (D)

What is in it — the surfaces

Surface What it is
Decisions The three strategic decisions, each linked to its evidence, framework, and dashboard
Evidence The register of claims, their confidence, and what still needs proving
Frameworks Five transparent scoring models with editable weights
Dashboards Three interactive tools that recompute from the data
Working Paper The long-form discussion behind each decision
Governance The standard (CIS), decision records, and testing standard

The engine (why the numbers are trustworthy)

  • All scoring/validation/evidence logic lives once in src/; dashboards import it, they do not reinvent it.
  • Weights live in YAML, not code — change a weight, re-render, the ranking moves.
  • 53 automated tests guard every calculation. A red suite is a hard stop.
  • All sample data is synthetic and labelled as such.

Honest status

v0.1, hypothesis stage. No segment, positioning, or GTM strategy is recommended yet. The pivotal next experiment is the AI-readiness pilot. See Current Status for the live picture and How to Use This Lab to start working with it.
