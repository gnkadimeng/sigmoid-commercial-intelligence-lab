Operationalising the proposal’s core claim — the CRM as the foundation for AI readiness

Dashboards › AI Readiness Scorecard

Supports the thesis H-CRM-AI — is the CRM the foundation for AI readiness? Context in Working Paper › AI Readiness.

Confidence D → C Synthetic data Related: Framework · Evidence Register

Illustrative, not validated. Scores use synthetic customer data and the AI Readiness Index, where data quality carries the heaviest weight. The underlying claim — that CRM/data maturity predicts AI value — is Hypothesis H-CRM-AI and is not yet validated. This scorecard exists to make that thesis measurable, so it can later be tested against real outcomes.

import sys, pathlib
_root = pathlib.Path.cwd()
while _root != _root.parent and not (_root / "src").is_dir():
    _root = _root.parent
sys.path.insert(0, str(_root))

import pandas as pd
import plotly.express as px
from IPython.display import HTML

from src.utils import load_sample
from src.scoring import calculate_ai_readiness_score, calculate_customer_fit_score

PLOTLY_TEMPLATE = "plotly_white"
Load customers and compute readiness + fit scores 
customers = load_sample("customer_profiles.csv").copy()
customers["ai_readiness_score"] = customers.apply(calculate_ai_readiness_score, axis=1)
customers["customer_fit_score"] = customers.apply(calculate_customer_fit_score, axis=1)

def band(score):
    if score >= 66:
        return "High"
    if score >= 40:
        return "Medium"
    return "Low"

customers["readiness_band"] = customers["ai_readiness_score"].map(band)
ranked = customers.sort_values("ai_readiness_score", ascending=False).reset_index(drop=True)
Build summary metrics (computed, not hardcoded) 
n = len(customers)
mean_readiness = customers["ai_readiness_score"].mean()
high_n = int((customers["readiness_band"] == "High").sum())
high_pct = 100 * high_n / n
top = ranked.iloc[0]

def metric(label, value, sub=""):
    sub_html = f'<div class="metric-sub">{sub}</div>' if sub else ""
    return (f'<div class="metric"><div class="metric-label">{label}</div>'
            f'<div class="metric-value">{value}</div>{sub_html}</div>')

HTML(
    '<div class="metric-row">'
    + metric("Customers scored", n, "synthetic profiles")
    + metric("Mean AI readiness", f"{mean_readiness:.1f}", "out of 100 (illustrative)")
    + metric("High-readiness share", f"{high_pct:.0f}%", f"{high_n} of {n} customers")
    + metric("Most ready", top["company_name"], f'score {top["ai_readiness_score"]:.1f}')
    + "</div>"
)

Interpretation

  • data_quality is the load-bearing input of this index because the proposal asserts AI value flows from the data/CRM foundation.
  • A wide readiness distribution is useful: it gives the eventual pilot enough spread to detect whether readiness actually predicts AI value.
  • Readiness and fit are scored independently — treat any apparent relationship in the scatter as a question to test, not a finding.
ImportantThe claim under test

The heavy data_quality weight makes this index a restatement of Hypothesis H-CRM-AI (grade D → C). Validating it requires linking readiness scores to actual AI outcomes in a pilot — see Working Paper Ch. 6.

WarningLimitations
  • Synthetic customers only; scores exercise the machinery, they do not describe a real book of business.
  • The index encodes the thesis rather than testing it — avoid circular reasoning when validating.

## Interpretation

- `data_quality` is the load-bearing input of this index because the proposal asserts AI value flows
  from the data/CRM foundation.
- A wide readiness distribution is *useful*: it gives the eventual pilot enough spread to detect
  whether readiness actually predicts AI value.
- Readiness and fit are scored independently — treat any apparent relationship in the scatter as a
  question to test, not a finding.

::: {.callout-important appearance="simple"}
## The claim under test
The heavy `data_quality` weight makes this index a restatement of **Hypothesis H-CRM-AI (grade
D → C)**. Validating it requires linking readiness scores to *actual* AI outcomes in a pilot — see
[Working Paper Ch. 6](../working-paper/06-ai-readiness.qmd).
:::

::: {.callout-warning appearance="simple"}
## Limitations
- Synthetic customers only; scores exercise the machinery, they do not describe a real book of
  business.
- The index *encodes* the thesis rather than testing it — avoid circular reasoning when validating.
:::
